Documentación Técnica

1. Introducción

Proyectorium es una aplicación de escritorio desarrollada en JavaFX para la gestión eficiente de un cine. Su objetivo es mejorar la experiencia tanto para los clientes como para los trabajadores del cine. Los clientes podrán consultar la cartelera, adquirir entradas electrónicas y obtener información de las películas, mientras que el personal contará con herramientas para la administración del cine, incluyendo películas, proveedores y categorías.

2. Objetivos

  • Crear una aplicación intuitiva y accesible para los clientes del cine.
  • Ofrecer a los empleados una plataforma para la gestión de datos clave del cine.

Objetivos Específicos

  • Gestión de categorías: Mostrar categorías con fecha de creación y clasificación por edades.
  • Gestión de proveedores: Mostrar información contractual y personal de proveedores.
  • Generación de entradas: Crear tickets electrónicos únicos y seguros.
  • Gestión de usuarios: Control de acceso con roles (administrador y vendedor).
  • Seguridad y validación: Validaciones para prevenir errores y fraudes.

2.2. Especificación de Casos de Uso (Jerarquizada)

2.2.1. Casos de uso - Gestión de Películas

  • Agregar película.
  • Editar película.
  • Eliminar película.
  • Asignar categoría y proveedor.
  • Consultar películas existentes.

2.2.2. Casos de uso - Gestión de Categorías

  • Crear nueva categoría.
  • Editar categoría.
  • Eliminar categoría.
  • Consultar categorías existentes.

2.2.3. Casos de uso - Gestión de Proveedores

  • Registrar proveedor.
  • Editar datos del proveedor.
  • Eliminar datos del proveedor.
  • Consultar contratos vigentes.

2.2.4. Casos de uso - Gestión de Entradas

  • Generar ticket electrónico.
  • Modificar ticket electrónico.
  • Eliminar ticket electrónico.
  • Consultar lista de tickets electrónicos.

2.2.5. Casos de uso - Gestión de Usuarios

  • Iniciar sesión con autenticación.
  • Registrar vendedor o administrador (solo por administrador).
  • Asignar roles.

2.3. Modelo conceptual de datos

Este apartado incluye un diagrama ER con las siguientes entidades principales:

  • Usuario (1) ── (N) Ticket
  • Película (1) ── (N) Ticket
  • Categorías (N) ── (M) Películas
  • Proveedor (1) ── (N) Películas

Atributos clave:

  • Película: título, duración, clasificación, sinopsis.
  • Categoría: nombre, edad mínima, fecha de creación.
  • Proveedor: nombre, contrato, inicio/fin, contacto.
  • Ticket: ID, código ,precio.
  • Usuario: Tipo, email.

2.4. Arquitectura de software de la solución

  • Arquitectura: Cliente-Servidor
  • Patrón de diseño: Modelo-Vista-Controlador (MVC)
  • Frontend: JavaFX
  • Backend: Java EE / RESTful Services
  • Base de datos: MySQL o SQLite
  • Seguridad: Hash de contraseñas, autenticación con roles
  • Comunicación: Cliente REST y servidor RESTful
  • Herramientas: Git, Maven, Scene Builder

3. Funcionalidades

Para Usuarios

  • Registro e inicio de sesión.
  • Consulta de cartelera.
  • Gestión de entradas (agregar, modificar, eliminar).

Para Administradores

  • Gestión de películas (agregar, modificar, eliminar).
  • Gestión de proveedores (agregar, modificar, eliminar).
  • Gestión de categorías (agregar, modificar, eliminar).

4. Tecnologías a Utilizar

  • Lenguaje: Java 8
  • Framework: JavaFX
  • Base de datos: MySQL / SQLite
  • Herramientas: Scene Builder, Maven, Git

5. Arquitectura del Sistema

  • Patrón de diseño: Modelo-Vista-Controlador (MVC)
  • Base de datos: Tablas para usuarios, películas, categorias, tickets y proveedores.
  • Seguridad: Hash de contraseñas y control por roles.

6. Conclusión

Proyectorium es una solución integral que busca mejorar la experiencia del cliente en el cine y optimizar la gestión operativa mediante tecnología moderna, segura y escalable.
